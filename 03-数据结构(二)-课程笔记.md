## 一、Trie 树（字典树）

### 1. 核心思想

Trie 是一棵**多叉树**，用来高效存储和查询字符串集合。

**关键理解**：
- 每条**从根到节点的路径**表示一个前缀
- 不是每个节点都代表一个单词，只有标记了 `cnt[]` 的才是
- 根节点是空的（编号0），不存任何字符

**yxc 画的 Trie 图**（从截图还原）：

```
        root(0)
       /    \
      a      c
     / \      \
    b   c      a
    |   |      |
    c*  d*     t*
```
插入 "abc"、"acd"、"cat" 后的样子。标 `*` 的是单词结尾（cnt > 0）。

### 2. 存储方式（数组模拟）

```
son[N][26]  — son[p][u] 表示节点 p 的第 u 个孩子的编号
cnt[N]      — cnt[p] 表示以节点 p 结尾的单词数量
idx         — 当前已分配的节点编号（0 是根，新节点从 1 开始）
```

**为什么用数组不用指针？** 竞赛中数组快、好写、不容易出错。

### 3. 模板代码

```cpp
int son[N][26], cnt[N], idx;

// 插入
void insert(char *str) {
    int p = 0;                      // 从根出发
    for (int i = 0; str[i]; i++) {
        int u = str[i] - 'a';       // 字符映射到 0~25
        if (!son[p][u]) son[p][u] = ++idx;  // 没路就建
        p = son[p][u];              // 走到下一层
    }
    cnt[p]++;                       // 标记单词结尾
}

// 查询
int query(char *str) {
    int p = 0;
    for (int i = 0; str[i]; i++) {
        int u = str[i] - 'a';
        if (!son[p][u]) return 0;   // 路断了 → 不存在
        p = son[p][u];
    }
    return cnt[p];                  // 返回该单词出现次数
}
```

### 4. 应用：最大异或对（AcWing 143）

**问题**：给定 n 个整数，找两个数使得异或值最大。

**思路**：
- 把每个数的**二进制表示**（从高位到低位，共31位）插入 Trie
- 对每个数，在 Trie 中贪心地走：每一位**尽量走相反的方向**（异或为1更大）

**为什么从高位开始？** 高位的 1 比低位所有位加起来都大（二进制权重）。

```cpp
int son[N * 31][2], idx;  // 二进制 Trie，每个节点只有 0/1 两个孩子

void insert(int x) {
    int p = 0;
    for (int i = 30; i >= 0; i--) {
        int u = x >> i & 1;
        if (!son[p][u]) son[p][u] = ++idx;
        p = son[p][u];
    }
}

int query(int x) {  // 找与 x 异或最大的值
    int p = 0, res = 0;
    for (int i = 30; i >= 0; i--) {
        int u = x >> i & 1;
        if (son[p][!u]) {           // 能走相反方向就走
            res += 1 << i;
            p = son[p][!u];
        } else {
            p = son[p][u];          // 不能就只好走同方向
        }
    }
    return res;
}
```

### 5. 易错点

- `son[N][26]` 的 N 要开到**所有字符串总长度之和**，不是字符串个数
- 二进制 Trie 的 N 要开到 `数的个数 × 31`
- 0 号节点既是根又代表"不存在"，所以新节点从 `++idx` 开始

## 二、并查集 (Union-Find)

### 1. 核心思想

用**树**来表示集合：每个集合是一棵树，树根就是集合的代表元素。

**支持的操作**：
1. **find(x)**：找 x 所在集合的根（代表元素）
2. **union(a, b)**：合并 a 和 b 所在的集合

**yxc 画的并查集图**（从截图还原）：

```
合并前：          合并后（把 A 的根接到 B 的根下）：
  1        4            4
 / \       |           /|\
2   3      5          5  1
                        / \
                       2   3
```

### 2. 核心优化：路径压缩

**问题**：如果树退化成链，find 就是 O(n) 的。

**解决**：find 的时候，顺手把路径上所有节点**直接指向根**。下次再查就是 O(1)。

```
压缩前：1 → 2 → 3 → 4(root)
压缩后：1 → 4, 2 → 4, 3 → 4
```

这就是 `p[x] = find(p[x])` 这行代码干的事——**一行代码，效果拔群**。

### 3. 三个版本的模板

#### 版本1：朴素版（最常用）

```cpp
int p[N];

int find(int x) {
    if (p[x] != x) p[x] = find(p[x]);  // 路径压缩
    return p[x];
}

// 初始化：每个人自成一派
for (int i = 1; i <= n; i++) p[i] = i;

// 合并 a 和 b
p[find(a)] = find(b);

// 判断同一集合
if (find(a) == find(b)) ...
```

#### 版本2：维护集合大小

```cpp
int p[N], sz[N];

int find(int x) {
    if (p[x] != x) p[x] = find(p[x]);
    return p[x];
}

// 初始化
for (int i = 1; i <= n; i++) p[i] = i, sz[i] = 1;

// ⚠️ 合并顺序很重要：先加 size，再改指向
sz[find(b)] += sz[find(a)];
p[find(a)] = find(b);
```

#### 版本3：维护到根的距离（带权并查集）

```cpp
int p[N], d[N];  // d[x] = x 到 p[x] 的距离

int find(int x) {
    if (p[x] != x) {
        int u = find(p[x]);      // 先递归找根
        d[x] += d[p[x]];         // 路径压缩时累加距离
        p[x] = u;                // 直接指向根
    }
    return p[x];
}
```

### 4. 经典题：食物链（AcWing 240）

**题意**：A 吃 B，B 吃 C，C 吃 A（三种动物形成环）。给一系列描述，判断哪些是假话。

**yxc 的思路**（从板书还原）：
- 用**到根节点的距离 d[x] % 3** 来表示关系：
  - `d[x] % 3 == 0`：x 和根是**同类**
  - `d[x] % 3 == 1`：x **吃**根
  - `d[x] % 3 == 2`：x **被**根吃

**判断逻辑**：
- "x 和 y 是同类"：如果已在同一集合，检查 `(d[x] - d[y]) % 3 == 0`
- "x 吃 y"：如果已在同一集合，检查 `(d[x] - d[y] - 1) % 3 == 0`
- 不在同一集合：直接合并，设置距离

### 5. 易错点

- 合并时 `p[find(a)] = find(b)` — 必须对**根**操作，不是对 a、b 本身
- 维护 size 时**先加后改**，顺序反了 size 就错了（因为 find 值会变）
- 带权并查集的 d[x] 在路径压缩后变成了到根的距离，不是到父节点的距离
- 食物链那题 C++ 要注意取模：`(a % 3 + 3) % 3`，防止负数

## 三、堆 (Heap)

### 1. 核心思想

堆 = **完全二叉树** + **堆序性质**（小根堆：每个节点 ≤ 子节点）。

**yxc 画的堆结构**（从截图还原）：

```
         1(根)
        / \
       2    3
      / \  / \
     4   5 6   7
    下标关系：
    父节点：i/2
    左孩子：2i
    右孩子：2i+1
```

**用一维数组存**：下标从 1 开始，`h[1]` 是根（最小值）。

### 2. 核心操作：down 和 up

**所有堆操作都基于这两个**：

```cpp
// down：把一个值"下沉"到正确位置
void down(int u) {
    int t = u;
    if (u * 2 <= size && h[u * 2] < h[t]) t = u * 2;        // 左孩子更小
    if (u * 2 + 1 <= size && h[u * 2 + 1] < h[t]) t = u * 2 + 1;  // 右孩子更小
    if (u != t) {
        swap(h[u], h[t]);  // 和最小的孩子交换
        down(t);            // 继续下沉
    }
}

// up：把一个值"上浮"到正确位置
void up(int u) {
    while (u / 2 && h[u] < h[u / 2]) {
        swap(h[u], h[u / 2]);
        u >>= 1;
    }
}
```

### 3. 五种堆操作

| 操作 | 实现 |
|------|------|
| 插入 | `h[++size] = x; up(size);` |
| 取最小值 | `h[1]` |
| 删除最小值 | `h[1] = h[size--]; down(1);` |
| 删除第 k 个 | `h[k] = h[size--]; down(k); up(k);` |
| 修改第 k 个 | `h[k] = x; down(k); up(k);` |

**删除的技巧**：把最后一个元素覆盖到要删的位置，然后 down+up 调整。不知道该 down 还是 up？**两个都调用**，只有一个会生效。

### 4. O(n) 建堆

```cpp
// 从 n/2 开始往前 down（叶子节点不用 down）
for (int i = n / 2; i; i--) down(i);
```

**为什么是 O(n) 不是 O(nlogn)？** 大部分节点在底层，down 的距离短。数学推导：总操作数 ≈ n。

### 5. 模拟堆（维护插入顺序，AcWing 839）

需要支持"删除第 k 个插入的元素"和"修改第 k 个插入的元素"：

```cpp
int h[N], ph[N], hp[N], cnt;
// ph[k] = 第 k 个插入的元素在堆中的位置
// hp[i] = 堆中位置 i 的元素是第几个插入的
// 三者关系：h[ph[k]] = 第k个元素的值，hp[ph[k]] = k

void heap_swap(int a, int b) {
    swap(ph[hp[a]], ph[hp[b]]);  // 更新映射
    swap(hp[a], hp[b]);
    swap(h[a], h[b]);
}
```

### 6. 易错点

- 下标从 **1** 开始，不是 0（否则父子关系公式不对）
- 删除最小值是把 `h[size]` 放到 `h[1]`，不是直接删
- 模拟堆的 `heap_swap` 必须同时维护 `ph[]` 和 `hp[]`，漏了就乱套

## 四、Hash 表

### 1. 核心思想

将**任意大范围**的数映射到**小范围**的数组下标里。

**哈希函数**：`h(x) = (x % N + N) % N`
- 为什么 `+N) % N`？处理负数。C++ 的 `%` 对负数取模可能返回负值。
- N 取**质数**且离 2 的整次幂尽量远，冲突最少（经验结论）。

### 2. 处理冲突：两种方法

#### 方法1：拉链法

每个槽位挂一个链表：

```
h[0] → 3 → 10 → null
h[1] → null
h[2] → 5 → null
...
```

```cpp
int h[N], e[N], ne[N], idx;

void insert(int x) {
    int k = (x % N + N) % N;
    e[idx] = x;
    ne[idx] = h[k];     // 头插法
    h[k] = idx++;
}

bool find(int x) {
    int k = (x % N + N) % N;
    for (int i = h[k]; i != -1; i = ne[i])
        if (e[i] == x) return true;
    return false;
}

// 初始化：memset(h, -1, sizeof h);
```

#### 方法2：开放寻址法

冲突了就往后找空位（**线性探测**）：

```cpp
int h[N];  // N 要开到数据范围的 2~3 倍

int find(int x) {
    int t = (x % N + N) % N;
    while (h[t] != null && h[t] != x) {
        t++;
        if (t == N) t = 0;  // 绕回去
    }
    return t;  // 返回 x 应该在的位置
}

// 初始化：memset(h, 0x3f, sizeof h);  // 用一个不可能出现的值作为 null
```

**yxc 更推荐开放寻址法**：代码短，好写。

### 3. 字符串哈希（AcWing 841）

把字符串看成 **P 进制数**，用 **unsigned long long** 自然溢出取模（相当于 mod 2^64）。

```
"ABCD" → A×P³ + B×P² + C×P + D
```

**核心公式**：求子串 `str[l~r]` 的哈希值：
```
h[r] - h[l-1] × P^(r-l+1)
```

**类比**：就像十进制数取某几位。比如 12345 的第2~4位 = 12345 - 1×10³ × ... 不对，其实就是高位减去多余的部分。

```cpp
typedef unsigned long long ULL;
ULL h[N], p[N];  // h[i] = str[1~i] 的哈希值，p[i] = P^i

// 初始化
p[0] = 1;
for (int i = 1; i <= n; i++) {
    h[i] = h[i - 1] * P + str[i];  // 前缀哈希
    p[i] = p[i - 1] * P;
}

// 取子串哈希
ULL get(int l, int r) {
    return h[r] - h[l - 1] * p[r - l + 1];
}
```

### 4. 关键参数

- **P = 131 或 13331**（经验值，冲突概率极低）
- 不需要手动取模，`unsigned long long` 溢出就是自动 mod 2^64
- **不要把任何字符映射到 0**（否则 "A" 和 "AA" 的哈希值一样）

### 5. 易错点

- 拉链法 `h[]` 要初始化为 -1（`memset(h, -1, sizeof h)`）
- 开放寻址法 `h[]` 要初始化为 0x3f3f3f3f（`memset(h, 0x3f, sizeof h)`）
- 开放寻址法的 N 要开到**数据范围的 2~3 倍**，否则冲突太多会 TLE
- 字符串哈希中字符不能映射到 0

## 总结速查

| 数据结构 | 核心操作 | 时间复杂度 | 典型题 |
|----------|---------|-----------|--------|
| Trie | insert/query | O(字符串长度) | 字符串统计、最大异或对 |
| 并查集 | find/union | 近O(1)（均摊） | 连通性、食物链 |
| 堆 | insert/top/delete/down/up | O(logn) | 堆排序、模拟堆 |
| Hash 表 | insert/find | O(1)（期望） | 数的存在性、字符串匹配 |
