## 零、图的存储（前置知识）

### 为什么要先学存储？

所有图论算法的第一步都是"怎么把图存下来"。存储方式选错，后面代码全废。

### 两种存储方式

#### 1. 邻接矩阵 `g[a][b]`

```
适用：点数少（N ≤ 1000）、稠密图
缺点：空间 O(N²)，点多就炸
用在：朴素 Dijkstra、Floyd、朴素 Prim
```

#### 2. 邻接表（数组模拟链表）

```cpp
int h[N], e[N], ne[N], idx;  // 每个点挂一个单链表

void add(int a, int b) {
    e[idx] = b, ne[idx] = h[a], h[a] = idx++;
}

// 初始化：memset(h, -1, sizeof h);
```

```
适用：点数多、稀疏图
本质：就是给每个点开一个链表，存它能到达的所有邻居
用在：几乎所有图论算法（DFS/BFS/堆优化Dijkstra/SPFA/Kruskal/二分图...）
```

**无向图**：每条边 a-b 存两次，`add(a,b)` 和 `add(b,a)`。

**带权图**：多开一个数组 `w[N]`，在 add 里加上 `w[idx] = c`。

### 记忆技巧

邻接表本质就是 **N 个单链表**，每个链表头是 `h[i]`，用的还是数据结构课学的那套数组模拟链表（`e[]` 存值，`ne[]` 存 next，`idx` 分配节点）。

## 一、DFS（深度优先搜索）

### 核心思想

一条路走到黑，走不动了再回头。像走迷宫一样，每到一个岔路口就选一条路往下走，走到死胡同就退回上一个岔路口换另一条路。

### 关键概念

- **回溯**：走不通了退回来，这是 DFS 的灵魂
- **剪枝**：提前判断某条路不可能得到答案，直接跳过（优化搜索速度）
- **st[] 数组**：标记哪些点/状态已经访问过，防止重复搜索

### 执行过程（以树为例）

```
      1
     / \
    2   3
   / \
  4   5

DFS 访问顺序：1 → 2 → 4 → (回溯) → 5 → (回溯) → (回溯) → 3
```

每到一个节点：
1. 标记为已访问
2. 遍历所有邻居
3. 对未访问的邻居递归 DFS
4. 全部邻居处理完后自动回溯（函数返回）

### 模板

```cpp
void dfs(int u) {
    st[u] = true;               // 标记已访问

    for (int i = h[u]; i != -1; i = ne[i]) {
        int j = e[i];           // j 是 u 的邻居
        if (!st[j]) dfs(j);     // 未访问过就递归
    }
}
```

### 经典应用：树的重心（AcWing 846）

**题意**：删除树中某个节点后，剩余各连通块中点数的最大值最小，求这个最大值。

**思路**：
- DFS 遍历整棵树，对每个节点 u 计算：删掉 u 后各部分的大小
- 删掉 u 后有两类部分：① u 的各个子树 ② 除了 u 的子树外剩下的部分（= n - 子树总大小 - 1）
- 取这些部分的最大值，所有节点的最大值中取最小

```cpp
int ans = n;  // 全局答案

int dfs(int u) {
    st[u] = true;
    int sum = 1, res = 0;  // sum: u 的子树大小, res: 删 u 后最大连通块大小

    for (int i = h[u]; i != -1; i = ne[i]) {
        int j = e[i];
        if (!st[j]) {
            int s = dfs(j);      // s = 子树 j 的大小
            res = max(res, s);   // 子树部分的最大值
            sum += s;
        }
    }

    res = max(res, n - sum);     // "上面"那部分的大小
    ans = min(ans, res);         // 更新全局答案
    return sum;
}
```

### DFS 的本质

DFS = **递归 + 回溯**。用系统栈来记录"来时的路"。空间复杂度 O(h)，h 是搜索树的深度。

### 易错点

- 忘记 `st[u] = true`，导致死循环
- 无向图忘记存两条边
- 递归层数太深（>1e5）可能爆栈，考虑手写栈或改 BFS

## 二、BFS（宽度优先搜索）

### 核心思想

一层一层扩展，先搜离起点最近的，再搜远的。像水波扩散一样。

### 关键性质

**BFS 天然求最短路**（前提：所有边权重相等，即边权为 1 的情况）。

因为 BFS 是按层扩展的，第一次到达某个点时，经过的层数就是最短距离。

### 执行过程

```
层0:  {1}
层1:  {2, 3}     ← 1 的邻居
层2:  {4, 5}     ← 2, 3 的邻居
```

### 模板

```cpp
int d[N];  // d[i] = 起点到 i 的距离，-1 表示未访问

int bfs() {
    memset(d, -1, sizeof d);
    queue<int> q;
    d[1] = 0;
    q.push(1);

    while (q.size()) {
        int t = q.front();
        q.pop();

        for (int i = h[t]; i != -1; i = ne[i]) {
            int j = e[i];
            if (d[j] == -1) {       // 未访问过
                d[j] = d[t] + 1;    // 距离 +1
                q.push(j);
            }
        }
    }
    return d[n];
}
```

### DFS vs BFS 选择

| 特点 | DFS | BFS |
|------|-----|-----|
| 数据结构 | 栈（递归） | 队列 |
| 空间 | O(h) 搜索深度 | O(w) 最大宽度 |
| 最短路 | ❌ 不保证 | ✅ 边权为1时保证 |
| 适用 | 全排列、组合、连通性、树形问题 | 最短路、层序遍历 |

**经验**：
- 要**最短/最少步数** → BFS
- 要**遍历所有方案** → DFS
- 都行的时候看数据范围：深度大用 BFS，宽度大用 DFS

## 三、拓扑排序

### 什么是拓扑序？

对有向无环图（DAG）的所有节点排序，使得对每条边 u→v，u 排在 v 前面。

**直觉理解**：选课顺序。A 是 B 的先修课，那 A 一定排在 B 前面。

### 关键概念

- **入度 d[i]**：有多少条边指向 i
- **DAG**：有向无环图。只有 DAG 才有拓扑序

### 算法步骤

1. 把所有入度为 0 的点入队（它们没有先修课，可以先选）
2. 取出队头 t，遍历 t 的所有出边 t→j
3. j 的入度减 1（t 已经"学完"了），如果 j 入度变成 0，入队
4. 如果最终所有点都入过队 → 存在拓扑序；否则有环

### 模板

```cpp
bool topsort() {
    int hh = 0, tt = -1;

    for (int i = 1; i <= n; i++)
        if (!d[i])
            q[++tt] = i;        // 入度为 0 的先入队

    while (hh <= tt) {
        int t = q[hh++];

        for (int i = h[t]; i != -1; i = ne[i]) {
            int j = e[i];
            if (--d[j] == 0)    // 入度减到 0 就入队
                q[++tt] = j;
        }
    }

    return tt == n - 1;          // 全部入队 = 有拓扑序
}
```

### 拓扑序的结果

排完后 `q[0..n-1]` 就是一个合法的拓扑序列。

### 易错点

- 拓扑排序只适用于 **有向图**
- 如果图有环，不存在拓扑序（return false）
- 拓扑序**不唯一**（多个入度为 0 的点，选谁先都行）

## 四、最短路算法（重点！）

### 算法选择决策树

这是 yxc 课上最重要的一张图——**看到最短路题，先判断用哪个算法**：

```
最短路问题
├── 单源最短路（一个起点到所有点）
│   ├── 所有边权 ≥ 0
│   │   ├── 稠密图（m ~ n²）→ 朴素 Dijkstra O(n²)
│   │   └── 稀疏图（m ~ n）  → 堆优化 Dijkstra O(mlogn)
│   └── 有负权边
│       ├── 无边数限制 → SPFA O(m)~O(nm)
│       └── 有边数限制（≤k条边）→ Bellman-Ford O(nm)
└── 多源最短路（任意两点间）→ Floyd O(n³)
```

**记住这个决策树，最短路问题就不会选错算法。**

### 4.1 朴素 Dijkstra

#### 适用条件
- 单源最短路
- **所有边权 ≥ 0**（有负权就不能用！）
- 稠密图（n ≤ 1000 左右）

#### 核心思想（贪心）

维护两个集合：
- S：已经确定最短路的点
- T：还没确定的点

每次从 T 中选**距离最小**的点 t，加入 S，然后用 t **更新**它的邻居的距离。

**为什么是对的？** 因为所有边权 ≥ 0，当前 T 中距离最小的那个点，不可能通过其他更远的点绕过来变得更近。

#### 执行过程

```
起点 1，求到所有点的最短路
初始：dist[1]=0, 其余=INF

第1轮：从T中选dist最小的 → 1(dist=0)
       用1更新邻居：dist[2]=2, dist[3]=5
       S={1}

第2轮：从T中选dist最小的 → 2(dist=2)
       用2更新邻居：dist[3]=min(5, 2+1)=3
       S={1,2}

第3轮：从T中选dist最小的 → 3(dist=3)
       S={1,2,3}
```

#### 模板

```cpp
int g[N][N];  // 邻接矩阵
int dist[N];
bool st[N];   // st[i]=true 表示 i 已在 S 中

int dijkstra() {
    memset(dist, 0x3f, sizeof dist);
    dist[1] = 0;

    for (int i = 0; i < n - 1; i++) {   // 循环 n-1 次
        // 1. 找 T 中 dist 最小的点
        int t = -1;
        for (int j = 1; j <= n; j++)
            if (!st[j] && (t == -1 || dist[t] > dist[j]))
                t = j;

        st[t] = true;                    // 2. 加入 S

        // 3. 用 t 更新邻居
        for (int j = 1; j <= n; j++)
            dist[j] = min(dist[j], dist[t] + g[t][j]);
    }

    if (dist[n] == 0x3f3f3f3f) return -1;
    return dist[n];
}
```

#### 时间复杂度

外层 O(n)，内层找最小 O(n) + 更新 O(n) = **O(n²)**

#### 易错点

- g[][] 初始化为 0x3f3f3f3f，自环 `g[i][i] = 0`
- 重边取 min：`g[a][b] = min(g[a][b], w)`
- 循环 `n-1` 次不是 `n` 次（最后一个点不需要再处理）

---

### 4.2 堆优化 Dijkstra

#### 为什么需要优化？

朴素版每次"找 T 中最小"要 O(n)，用**小根堆**可以优化到 O(logn)。

#### 适用条件
- 单源、非负权、**稀疏图**（n 大，m 不大）

#### 模板

```cpp
typedef pair<int, int> PII;  // {距离, 节点编号}

int dijkstra() {
    memset(dist, 0x3f, sizeof dist);
    dist[1] = 0;
    priority_queue<PII, vector<PII>, greater<PII>> heap;  // 小根堆
    heap.push({0, 1});

    while (heap.size()) {
        auto [distance, ver] = heap.top();
        heap.pop();

        if (st[ver]) continue;   // 已确定最短路，跳过（冗余元素）
        st[ver] = true;

        for (int i = h[ver]; i != -1; i = ne[i]) {
            int j = e[i];
            if (dist[j] > distance + w[i]) {
                dist[j] = distance + w[i];
                heap.push({dist[j], j});  // 可能有重复，但不影响正确性
            }
        }
    }

    if (dist[n] == 0x3f3f3f3f) return -1;
    return dist[n];
}
```

#### 关键理解

- 堆里可能有**同一个点的多个版本**（每次更新距离都 push 一个新的）
- 所以 pop 出来后要检查 `if (st[ver]) continue`，跳过已经处理过的
- 这比手写"修改堆中元素"简单得多，虽然堆大小可能到 O(m)，但总复杂度还是 **O(mlogn)**

#### 易错点

- PII 的 first 是**距离**，second 是**节点编号**（因为 pair 默认按 first 排序）
- 用 `greater<PII>` 才是小根堆（默认是大根堆）
- 邻接表带权：`w[i]` 是第 i 条边的权重

### 4.3 Bellman-Ford

#### 适用条件

- 单源最短路
- **可以有负权边**
- 特别适合：**限制最多经过 k 条边**的最短路

#### 核心思想

暴力松弛：对所有边重复松弛 n-1 次（因为最短路最多 n-1 条边）。

**松弛操作**：如果 `dist[b] > dist[a] + w`，就更新 `dist[b] = dist[a] + w`

#### 模板

```cpp
struct Edge { int a, b, w; } edges[M];
int dist[N], backup[N];

int bellman_ford() {
    memset(dist, 0x3f, sizeof dist);
    dist[1] = 0;

    for (int i = 0; i < k; i++) {        // k 次迭代 = 最多 k 条边
        memcpy(backup, dist, sizeof dist); // ⚠️ 备份！防止串联更新
        for (int j = 0; j < m; j++) {
            int a = edges[j].a, b = edges[j].b, w = edges[j].w;
            dist[b] = min(dist[b], backup[a] + w);  // 用 backup 不是 dist
        }
    }

    if (dist[n] > 0x3f3f3f3f / 2) return -1;  // 注意是 /2 不是 ==
    return dist[n];
}
```

#### 为什么要备份 (backup)？

如果不备份，同一轮迭代中，a 更新了 b，b 又更新了 c，相当于用了两条边，但我们可能只允许用 1 条边。备份后每轮只用上一轮的结果更新，保证**恰好用 i 条边**。

#### 为什么判断用 `> 0x3f3f3f3f / 2` 而不是 `==`？

因为有负权边，INF 可能被"稍微减小"一点（比如 INF + (-1) = INF - 1），但仍然不可达。所以用 `/2` 留个余量。

#### 时间复杂度

O(nm) — 简单粗暴，n 轮 × m 条边

### 4.4 SPFA（最常用！）

#### 核心思想

Bellman-Ford 的**队列优化**。观察：只有 `dist[a]` 变小了，`dist[b]` 才可能被 a 更新。所以只把"距离变小的点"放入队列，避免无意义的松弛。

#### 适用条件

- 单源最短路
- 可以有负权边
- **一般情况下比 Bellman-Ford 快很多**（平均 O(m)）
- 最坏 O(nm)，被故意卡时退化

#### 模板

```cpp
int spfa() {
    memset(dist, 0x3f, sizeof dist);
    dist[1] = 0;

    queue<int> q;
    q.push(1);
    st[1] = true;       // st[i] = i 是否在队列中

    while (q.size()) {
        int t = q.front();
        q.pop();
        st[t] = false;  // 出队后标记为不在队列中

        for (int i = h[t]; i != -1; i = ne[i]) {
            int j = e[i];
            if (dist[j] > dist[t] + w[i]) {
                dist[j] = dist[t] + w[i];
                if (!st[j]) {       // 不在队列中才入队
                    q.push(j);
                    st[j] = true;
                }
            }
        }
    }

    if (dist[n] == 0x3f3f3f3f) return -1;
    return dist[n];
}
```

#### SPFA 判负环

增加 `cnt[j]` 记录从起点到 j 经过了多少条边。如果 `cnt[j] >= n`，说明路径上有 n+1 个点，由**抽屉原理**必有重复点 → 有环。

```cpp
bool spfa() {
    queue<int> q;
    for (int i = 1; i <= n; i++) {  // ⚠️ 所有点都入队（因为负环可能不从1可达）
        q.push(i);
        st[i] = true;
    }

    while (q.size()) {
        int t = q.front(); q.pop(); st[t] = false;

        for (int i = h[t]; i != -1; i = ne[i]) {
            int j = e[i];
            if (dist[j] > dist[t] + w[i]) {
                dist[j] = dist[t] + w[i];
                cnt[j] = cnt[t] + 1;
                if (cnt[j] >= n) return true;  // 有负环！
                if (!st[j]) { q.push(j); st[j] = true; }
            }
        }
    }
    return false;
}
```

#### SPFA vs Dijkstra

| | Dijkstra | SPFA |
|---|---------|------|
| 负权 | ❌ | ✅ |
| 一般速度 | O(mlogn) 稳定 | O(m) 平均，但可能被卡 |
| 实战建议 | 非负权首选 | 有负权必须用 |

**注意**：很多竞赛题会故意卡 SPFA，所以非负权的情况下优先用 Dijkstra。

### 4.5 Floyd

#### 适用条件
- **多源最短路**（任意两点间的最短距离）
- n ≤ 200~300（O(n³) 限制）

#### 核心思想（动态规划）

`d[k][i][j]` = 只经过前 k 个点作为中间节点，i 到 j 的最短距离

状态转移：`d[k][i][j] = min(d[k-1][i][j], d[k-1][i][k] + d[k-1][k][j])`

第一维 k 可以压缩掉 → 三重循环，**k 必须在最外层**。

#### 模板

```cpp
// 初始化
for (int i = 1; i <= n; i++)
    for (int j = 1; j <= n; j++)
        if (i == j) d[i][j] = 0;
        else d[i][j] = INF;

// 读入边：d[a][b] = min(d[a][b], w);

void floyd() {
    for (int k = 1; k <= n; k++)          // ⚠️ k 在最外层！
        for (int i = 1; i <= n; i++)
            for (int j = 1; j <= n; j++)
                d[i][j] = min(d[i][j], d[i][k] + d[k][j]);
}
```

#### 易错点

- **k 必须在最外层**，写反了就全错
- 自环 `d[i][i] = 0`
- 查询时判不可达：`if (d[a][b] > INF / 2)` 不能用 `== INF`（中间可能被负权边影响）

## 五、最小生成树

### 什么是最小生成树？

在无向连通图中，选 n-1 条边连接所有 n 个点，使总边权最小。

### 算法选择

```
最小生成树
├── 稠密图（m ~ n²）→ 朴素 Prim O(n²)
└── 稀疏图（m ~ n）  → Kruskal O(mlogm)
```

### 5.1 朴素 Prim

#### 核心思想

和 Dijkstra **几乎一模一样**！区别在于：
- Dijkstra：`dist[j]` = 起点到 j 的距离
- Prim：`dist[j]` = j 到**已选集合**的最短距离

#### 执行过程

```
1. 初始：所有点的 dist = INF
2. 每次选 dist 最小的点 t 加入集合
3. 用 t 更新剩余点到集合的距离：dist[j] = min(dist[j], g[t][j])
4. 重复 n 次
```

#### 模板

```cpp
int prim() {
    memset(dist, 0x3f, sizeof dist);
    int res = 0;

    for (int i = 0; i < n; i++) {
        int t = -1;
        for (int j = 1; j <= n; j++)
            if (!st[j] && (t == -1 || dist[t] > dist[j]))
                t = j;

        if (i && dist[t] == INF) return INF;  // 不连通
        if (i) res += dist[t];  // ⚠️ 先累加再更新（防止自环影响）
        st[t] = true;

        for (int j = 1; j <= n; j++)
            dist[j] = min(dist[j], g[t][j]);  // 注意是 g[t][j] 不是 dist[t]+g[t][j]
    }
    return res;
}
```

#### Prim vs Dijkstra 的区别（一行之差）

```
Dijkstra: dist[j] = min(dist[j], dist[t] + g[t][j])  // 起点到 j
Prim:     dist[j] = min(dist[j], g[t][j])             // 集合到 j
```

### 5.2 Kruskal

#### 核心思想

1. 把所有边按权重**从小到大排序**
2. 依次取出每条边，如果这条边的两个端点**不在同一集合**（不连通），就选它
3. 判断连通性用**并查集**

#### 模板

```cpp
struct Edge {
    int a, b, w;
    bool operator< (const Edge &W) const { return w < W.w; }
} edges[M];

int p[N];
int find(int x) { if (p[x] != x) p[x] = find(p[x]); return p[x]; }

int kruskal() {
    sort(edges, edges + m);
    for (int i = 1; i <= n; i++) p[i] = i;

    int res = 0, cnt = 0;
    for (int i = 0; i < m; i++) {
        int a = find(edges[i].a), b = find(edges[i].b), w = edges[i].w;
        if (a != b) {
            p[a] = b;
            res += w;
            cnt++;
        }
    }

    if (cnt < n - 1) return INF;  // 不连通
    return res;
}
```

#### 为什么 Kruskal 是对的？

贪心：每次选最短的、不形成环的边。可以用**切割定理**证明：对任意切割，最短的跨切割边一定在某棵 MST 中。

#### Kruskal 的优势

- 代码简单（排序 + 并查集）
- 不需要建邻接表，直接用边数组
- 对稀疏图效率高

## 六、二分图

### 什么是二分图？

能把所有点分成两组，使得**所有边都在两组之间**（同组内没有边）。

### 判定定理

**一个图是二分图 ⟺ 图中不含奇数环**

### 6.1 染色法判定（AcWing 860）

#### 思路

用 DFS 交替染黑白两色。如果染色过程中发现邻居已经是相同颜色 → 不是二分图。

#### 模板

```cpp
int color[N];  // -1 未染色, 0 白, 1 黑

bool dfs(int u, int c) {
    color[u] = c;

    for (int i = h[u]; i != -1; i = ne[i]) {
        int j = e[i];
        if (color[j] == -1) {
            if (!dfs(j, !c)) return false;   // 邻居染相反色
        }
        else if (color[j] == c) return false; // 同色 = 矛盾
    }
    return true;
}

bool check() {
    memset(color, -1, sizeof color);
    for (int i = 1; i <= n; i++)
        if (color[i] == -1)        // 可能有多个连通分量
            if (!dfs(i, 0)) return false;
    return true;
}
```

### 6.2 匈牙利算法（最大匹配，AcWing 861）

#### 什么是最大匹配？

在二分图中，找最多的边，使得每个点最多被一条边覆盖。

**直觉**：相亲配对，左边一组人右边一组人，每人最多配一个对象，求最多能配多少对。

#### 核心思想

对左边每个人 x，尝试匹配：
1. 遍历 x 能匹配的所有右边的人 j
2. 如果 j 没人要 → 直接配
3. 如果 j 已经被 match[j] 占了 → 尝试让 match[j] 换一个人（递归）
4. 如果 match[j] 换成功了 → j 就归 x 了

这个"让别人换"的过程就是匈牙利算法的精髓——**增广路**。

#### 模板

```cpp
int match[N];  // match[j] = 右边 j 配对的左边的人（0 = 没配对）
bool st[N];    // 每轮匹配时，右边 j 是否被访问过

bool find(int x) {
    for (int i = h[x]; i != -1; i = ne[i]) {
        int j = e[i];
        if (!st[j]) {
            st[j] = true;
            if (match[j] == 0 || find(match[j])) {  // 没人要 或 那人能换
                match[j] = x;
                return true;
            }
        }
    }
    return false;
}

// 主函数
int res = 0;
for (int i = 1; i <= n1; i++) {
    memset(st, false, sizeof st);  // ⚠️ 每轮都要清空
    if (find(i)) res++;
}
```

#### 易错点

- `st[]` 每轮匹配**必须清空**（否则同一轮中一个右边点会被考虑多次）
- `match[j]` 存的是右边 j 对应的**左边**的人
- 只需要从左到右的边，不需要存反向边
